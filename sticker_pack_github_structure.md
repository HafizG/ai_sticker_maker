# AI Sticker Maker — Complete Architecture

## Overview

Android Kotlin Jetpack Compose sticker app. GitHub Pages as static CDN. Single `_master.json` drives everything. Categories are independent dynamic tabs (10-20). Packs belong to one or more categories. Region-specific + global packs. Extra/common section as a special tab. Fully offline after download.

### Key Points

- `"extra"` is just another category — no special handling needed
- Extra packs can be global (`"*"`) or region-specific (`"PK"`, `"SA"`)
- A pack can belong to multiple categories (`["funny", "religious"]`)
- `v` — bump this integer whenever you change anything
- `regions` array — just country codes, no names/flags
- `zones` — fallback groups for countries without dedicated region files

### Fields Reference

| Field | Required | Rules |
|---|---|---|
| `v` | yes | integer, bump on every change |
| `categories[].id` | yes | unique string, used as filter key |
| `categories[].name` | yes | display name shown as tab label |
| `regions[]` | yes | ISO 3166-1 alpha-2 country codes |
| `zones[].id` | yes | starts with `_`, used as filename |
| `zones[].countries` | yes | array of country codes this zone covers |
| `packs[].id` | yes | must match folder name under `packs/` |
| `packs[].name` | yes | display name shown in app |
| `packs[].cat` | yes | array of category IDs (1 or more) |
| `packs[].regions` | yes | array of country codes, zone IDs, or `"*"` for global |

### Region Targeting Rules

| Value | Meaning |
|---|---|
| `["PK"]` | Pakistan only |
| `["SA", "AE"]` | Saudi + UAE |
| `["SA", "_arab"]` | Saudi + all Arab fallback countries |
| `["*"]` | Every region, every zone, `_default` |
| `["_default"]` | Only in `_default.json` (countries with no match) |

## Pack Folder Convention

```
packs/{pack-id}/
├── tray_icon.webp          ← REQUIRED
├── 1.webp                  ← REQUIRED (at least 1)
├── 2.webp
├── 3.webp
└── ... up to 30.webp       ← max 30 per pack
```

| Rule | Detail |
|---|---|
| `tray_icon.webp` | Required. CI fails without it. 96x96px recommended. |
| Sticker naming | `1.webp`, `2.webp`, `3.webp` ... sequential, no gaps. |
| Max per pack | 30 stickers |
| Format | WebP with transparency (lossy with alpha) |
| Size per sticker | 512x512px recommended, under 100KB each |
| `count` | You never write this. CI scans folder and adds it to region JSON. |

---

## URL Convention

All URLs derived from pattern. Zero URLs stored in JSON.

| What | Pattern |
|---|---|
| Index | `{baseUrl}/index.json` |
| Region data | `{baseUrl}/regions/{CODE}.json` |
| Zone data | `{baseUrl}/regions/{zoneId}.json` |
| Default fallback | `{baseUrl}/regions/_default.json` |
| Pack tray icon | `{baseUrl}/packs/{packId}/tray_icon.webp` |
| Sticker image | `{baseUrl}/packs/{packId}/{n}.webp` |

---

## Fallback Chain

```
Device country detected → "LB" (Lebanon)

Step 1: Is "LB" in index.regions?
        → NO

Step 2: Is "LB" in any zone's countries?
        → YES → found in zones._arab
        → Fetch: regions/_arab.json ✅

Step 3: (if step 2 also failed)
        → Fetch: regions/_default.json ✅
```

| User Country | Resolves To | File |
|---|---|---|
| PK (Pakistan) | Direct region | `PK.json` |
| LB (Lebanon) | Zone: `_arab` | `_arab.json` |
| NP (Nepal) | Zone: `_south-asia` | `_south-asia.json` |
| DE (Germany) | Zone: `_eu` | `_eu.json` |
| JP (Japan) | Default | `_default.json` |

No user ever sees an empty app.

## CI Script: scripts/generate.js

```javascript
const fs = require('fs');
const path = require('path');

const master = JSON.parse(fs.readFileSync('sticker-cdn/_master.json', 'utf8'));
const packsDir = path.join('sticker-cdn', 'packs');
const regionsDir = path.join('sticker-cdn', 'regions');

if (!fs.existsSync(regionsDir)) fs.mkdirSync(regionsDir, { recursive: true });

let errors = [];

// ── Step 1: Scan pack folders ──
const packCounts = {};

for (const pack of master.packs) {
  const packPath = path.join(packsDir, pack.id);

  if (!fs.existsSync(packPath)) {
    errors.push(`Missing folder: packs/${pack.id}/`);
    continue;
  }

  if (!fs.existsSync(path.join(packPath, 'tray_icon.webp'))) {
    errors.push(`Missing tray_icon.webp in packs/${pack.id}/`);
  }

  const stickers = fs.readdirSync(packPath)
    .filter(f => f !== 'tray_icon.webp' && f.endsWith('.webp'))
    .length;

  if (stickers === 0) {
    errors.push(`No stickers in packs/${pack.id}/`);
    continue;
  }

  if (stickers > 30) {
    errors.push(`Too many stickers in packs/${pack.id}/ (${stickers}, max 30)`);
  }

  // Validate sequential naming: 1.webp, 2.webp, ...
  for (let i = 1; i <= stickers; i++) {
    if (!fs.existsSync(path.join(packPath, `${i}.webp`))) {
      errors.push(`Missing ${i}.webp in packs/${pack.id}/ (gap in sequence)`);
    }
  }

  // Validate all cat IDs exist
  const catIds = master.categories.map(c => c.id);
  for (const cat of pack.cat) {
    if (!catIds.includes(cat)) {
      errors.push(`Unknown category "${cat}" in pack ${pack.id}`);
    }
  }

  packCounts[pack.id] = stickers;
}

if (errors.length > 0) {
  console.error('\nValidation failed:\n');
  errors.forEach(e => console.error('  ' + e));
  process.exit(1);
}

// ── Step 2: Build targets ──
const targets = [
  ...master.regions.map(code => ({ id: code })),
  ...master.zones.map(z => ({ id: z.id })),
  { id: '_default' }
];

// ── Step 3: Generate region/zone files ──
for (const target of targets) {
  const filtered = master.packs
    .filter(p => p.regions.includes(target.id) || p.regions.includes('*'))
    .filter(p => packCounts[p.id])
    .map(({ regions, ...rest }) => ({ ...rest, count: packCounts[rest.id] }));

  fs.writeFileSync(
    path.join(regionsDir, `${target.id}.json`),
    JSON.stringify({ v: master.v, packs: filtered })
  );
}

// ── Step 4: Generate index.json ──
const regions = {};
master.regions.forEach(code => { regions[code] = master.v; });

const zones = {};
master.zones.forEach(z => {
  zones[z.id] = { v: master.v, countries: z.countries };
});

fs.writeFileSync(
  path.join('sticker-cdn', 'index.json'),
  JSON.stringify({
    v: master.v,
    baseUrl: 'https://hafizwaqas-me.github.io/practice/sticker-cdn',
    categories: master.categories,
    regions,
    zones,
    defaultRegion: '_default'
  })
);

console.log(`\nv${master.v} generated:`);
console.log(`  ${targets.length} region/zone files`);
console.log(`  ${master.packs.length} packs validated`);
console.log(`  index.json created\n`);
```

## Android App Architecture

### Tech Stack

| Concern | Library |
|---|---|
| UI | Jetpack Compose + Material 3 |
| Image loading | Coil 3 (Compose-native, WebP, disk cache) |
| Networking | Ktor Client |
| JSON parsing | Kotlinx Serialization |
| Local metadata | DataStore Preferences |
| File storage | `Context.filesDir` (no permissions) |
| DI | Koin or Hilt |
| Country detection | `TelephonyManager` / `Locale` |
| Sharing | FileProvider + `Intent.ACTION_SEND` |

### Country Detection

```
Priority:
  1. TelephonyManager.networkCountryIso  → SIM country (most reliable)
  2. Locale.getDefault().country          → device setting
  3. User manual selection                → settings fallback

All work offline. Detect once → save to DataStore → reuse.
```

### App Data Flow

```
FIRST LAUNCH:
  1. Fetch index.json                            (~2KB)
  2. Detect country → PK
  3. Fetch regions/PK.json                       (~5KB)
  4. Cache both to internal storage
  5. Save versions to DataStore
  6. Show categories + packs
  Total: ~7KB network

SUBSEQUENT LAUNCH:
  1. Fetch index.json                            (~2KB)
  2. Compare: index.v == cached? AND PK.v == cached?
     YES → use cached region JSON               (0KB extra)
     NO  → re-fetch regions/PK.json             (~5KB)
  Total: ~2KB network (cache hit) or ~7KB (cache miss)

NO NETWORK:
  1. Read cached index.json from files/cache/
  2. Read cached region JSON from files/cache/
  3. Everything works offline ✅

DOWNLOAD A PACK:
  Pack pk-funny-urdu, count: 3
  1. Fetch tray_icon.webp → save to files/stickers/pk-funny-urdu/
  2. Fetch 1.webp → save
  3. Fetch 2.webp → save
  4. Fetch 3.webp → save
  5. Update DataStore: downloaded_packs += "pk-funny-urdu"

DELETE A PACK:
  1. Delete folder files/stickers/pk-funny-urdu/
  2. Update DataStore: downloaded_packs -= "pk-funny-urdu"

SHARE A STICKER:
  1. Read from files/stickers/{packId}/{n}.webp
  2. Expose via FileProvider
  3. Share via Intent.ACTION_SEND
```

### Category Filtering Logic (App-Side)

```
All packs from region JSON are in ONE list.
App filters client-side based on selected tab.

allPacks = regionJson.packs   // e.g. PK.json has 50 packs

// User selects "Funny" tab
funnyPacks = allPacks.filter { "funny" in it.cat }

// User selects "Extra ★" tab
extraPacks = allPacks.filter { "extra" in it.cat }

// Show only tabs that have at least 1 pack
visibleCategories = categories.filter { cat ->
  allPacks.any { cat.id in it.cat }
}
```

### Empty Category Handling

```
Categories come from index.json (dynamic, 10-20).
Not every region will have packs for every category.

Example: PK might have 0 packs with cat: ["trending"]

Options (pick one in app):
  A) Hide empty tabs entirely (recommended)
  B) Show tab but display "Coming soon" message
  C) Show tab with global/extra packs as suggestions
```

## How to Add Content

### Add a new sticker pack

```
1. Create folder: sticker-cdn/packs/{pack-id}/
2. Add: tray_icon.webp + 1.webp, 2.webp ... (max 30)
3. Add to _master.json → packs[]:
   { "id": "{pack-id}", "name": "Pack Name", "cat": ["funny"], "regions": ["PK"] }
4. Bump "v" in _master.json
5. git push
```

### Add a new category

```
1. Add to _master.json → categories[]:
   { "id": "memes", "name": "Memes" }
2. Add "memes" to cat[] of relevant packs
3. Bump "v"
4. git push
```

### Add a new country (dedicated)

```
1. Add to _master.json → regions[]: "MA"
2. Add "MA" to regions[] of packs you want available there
3. Bump "v"
4. git push → CI creates regions/MA.json automatically
```

### Add a new zone

```
1. Add to _master.json → zones[]:
   { "id": "_central-asia", "countries": ["KZ", "UZ", "TM"] }
2. Add "_central-asia" to regions[] of relevant packs
3. Bump "v"
4. git push → CI creates regions/_central-asia.json automatically
```

### Make a pack global

```
Set regions to ["*"]
It appears in every region file, every zone file, and _default.
```

### Add extra/common stickers

```
Global extra:
  { "id": "extra-vintage", "name": "Vintage", "cat": ["extra"], "regions": ["*"] }

Region-specific extra:
  { "id": "extra-pk-14aug", "name": "14 August", "cat": ["extra"], "regions": ["PK"] }

Both are just regular packs with cat: ["extra"]. No special handling.
```

### Remove a category

```
1. Remove from _master.json → categories[]
2. Remove that category ID from all packs' cat[] arrays
3. Bump "v"
4. git push → app will no longer show that tab
```

| Metric | Value |
|---|---|

| Files you maintain | 1 (`_master.json`) + pack images |
| Files CI generates | 1 `index.json` + ~15 region/zone JSONs |
