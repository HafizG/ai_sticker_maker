# AI Sticker Maker — CDN Repository (v2)

Static sticker CDN served via GitHub Pages. Single `_master.json` drives everything.  
**Country-first architecture** — one request per country, no zones/wildcards.

## Repo Structure

```
├── .github/workflows/
│   └── deploy-stickers.yml    ← CI: validate, generate, deploy
│
├── scripts/
│   ├── generate.js            ← CI build script (auto-generates all JSONs)
│   └── create-dummy-webps.js  ← One-time: creates placeholder images
│
├── sticker-cdn/               ← Deployed to GitHub Pages
│   ├── _master.json           ← ⭐ ONLY file you edit manually
│   ├── index.json             ← Auto-generated (country list)
│   ├── index.html             ← Landing page
│   ├── countries/             ← Auto-generated (one per country)
│   │   ├── PK.json
│   │   ├── SA.json
│   │   ├── US.json
│   │   ├── _default.json
│   │   └── ... (51 files)
│   └── packs/                 ← You add sticker images here
│       ├── pk-funny-urdu/
│       │   ├── 1.webp
│       │   └── 2.webp
│       └── ...
│
└── README.md
```

## What You Touch

| File | Action |
|---|---|
| `sticker-cdn/_master.json` | Add/edit categories, packs, countries |
| `sticker-cdn/packs/{id}/` | Drop numbered `.webp` stickers |
| Bump `"v"` in `_master.json` | Every time you change anything |

**Everything else is auto-generated by CI.**

## Quick Start

### 1. Add a new sticker pack

```bash
# 1. Create the pack folder with images
mkdir sticker-cdn/packs/my-new-pack
# Add: 1.webp, 2.webp, ... (max 30)

# 2. Register in _master.json → packs
"my-new-pack": { "name": "My New Pack", "cat": ["funny"], "hidden": [] }

# 3. Add pack ID to relevant countries
"PK": { "categories": [...], "packs": [..., "my-new-pack"] }

# 4. Bump "v" → push
git add . && git commit -m "add my-new-pack" && git push
```

CI automatically: validates → generates country JSONs → deploys to GitHub Pages.

### 2. Add a new category

```jsonc
// In _master.json → categories
"memes": "Memes"

// Then add "memes" to cat[] of relevant packs
// Add "memes" to categories[] of relevant countries
// Bump "v" → push
```

### 3. Add a new country

```jsonc
// In _master.json → countries
"MA": {
  "categories": ["funny", "greetings", "love"],
  "packs": ["global-emoji-remix", "global-reactions", "global-love-hearts"]
}
// Or reuse another country's config:
"MA": { "same_as": "AE" }

// Bump "v" → push → CI creates countries/MA.json
```

### 4. Toggle a pack ON/OFF for a country

```jsonc
// Just add or remove the pack ID from the country's packs array
"PK": {
  "packs": ["pk-funny-urdu", "my-new-pack"]  // ← added
}
```

### 5. Hide a single sticker (without deleting the pack)

```jsonc
// In _master.json → packs → hidden array
"pk-funny-urdu": { "name": "...", "cat": ["funny"], "hidden": ["5.webp"] }
// Sticker 5.webp won't appear in any country, but the pack and other stickers remain
```

### 6. Control which categories a country sees

```jsonc
// Each country defines its own visible categories
"SA": {
  "categories": ["funny", "religious", "love"],  // ← no "sports" tab
  "packs": [...]
}
```

## Image Rules

| Rule | Spec |
|---|---|
| Sticker naming | `1.webp`, `2.webp`, `3.webp` ... |
| Max per pack | 30 stickers |
| Format | WebP with transparency |
| Size | 512×512px recommended, under 100KB each |
| Tray icon | First sticker auto-used as tray (no separate file needed) |

## CI Validation (auto-enforced)

| Check | Result if fails |
|---|---|
| Pack folder missing | ❌ Deploy blocked |
| Zero visible stickers | ❌ Deploy blocked |
| More than 30 stickers | ❌ Deploy blocked |
| Unknown category ID in pack | ❌ Deploy blocked |
| Unknown pack ID in country | ❌ Deploy blocked |
| Unknown category in country | ❌ Deploy blocked |
| Circular `same_as` pointer | ❌ Deploy blocked |
| Missing `_default` country | ❌ Deploy blocked |
| Hidden file doesn't exist | ⚠️ Warning |

## API Endpoints (for Android app)

Base URL: `https://hafizg.github.io/ai_sticker_maker`

| Endpoint | Description |
|---|---|
| `/index.json` | Country list + version |
| `/countries/PK.json` | **Everything for Pakistan** (categories, packs, sticker lists) |
| `/countries/_default.json` | Fallback for unlisted countries |
| `/packs/{id}/{n}.webp` | Sticker image |

### App Flow (1 request)

```
1. App knows user country: "PK"
2. GET /countries/PK.json → { categories, packs with stickers }
3. Build image URL: baseUrl + "/packs/" + pack.id + "/" + sticker
4. Done. No fallback chain needed.

For unlisted countries:
   GET /countries/JP.json → 404
   GET /countries/_default.json → ✅
```

## _master.json Reference (v2)

```jsonc
{
  "v": 2,                                // ← bump on every change
  "baseUrl": "https://hafizg.github.io/ai_sticker_maker",

  "categories": {                         // ← simple key-value
    "funny": "Funny",
    "religious": "Religious"
  },

  "packs": {                              // ← pack definitions
    "pk-funny-urdu": {
      "name": "Funny Urdu Stickers",      // ← display name
      "cat": ["funny"],                   // ← 1+ category IDs
      "hidden": ["5.webp"]               // ← hide specific stickers (or [])
    }
  },

  "countries": {                          // ← who sees what
    "_default": {                         // ← fallback (required)
      "categories": ["funny", "love"],
      "packs": ["global-emoji-remix"]
    },
    "PK": {                               // ← full definition
      "categories": ["funny", "religious", "love"],
      "packs": ["pk-funny-urdu", "global-emoji-remix"]
    },
    "LB": { "same_as": "AE" }            // ← alias (gets AE's config)
  }
}
```

### Country entry types

| Type | Example | Meaning |
|---|---|---|
| Full | `"PK": { "categories": [...], "packs": [...] }` | Custom config for Pakistan |
| Alias | `"LB": { "same_as": "AE" }` | Lebanon gets same as UAE |
| `_default` | `"_default": { ... }` | Fallback for unlisted countries |
