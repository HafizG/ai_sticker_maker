## What You Touch

| File | Action |
|---|---|
| `sticker-cdn/_master.json` | Add/edit categories, packs, regions, zones |
| `sticker-cdn/packs/{id}/` | Drop `tray_icon.webp` + numbered `.webp` stickers |
| Bump `"v"` in `_master.json` | Every time you change anything |

**Everything else is auto-generated by CI.**

## Quick Start

### 1. Add a new sticker pack

```bash
# 1. Create the pack folder with images
mkdir sticker-cdn/packs/my-new-pack
# Add: tray_icon.webp, 1.webp, 2.webp, ... (max 30)

# 2. Register in _master.json в†’ packs[]
{
  "id": "my-new-pack",
  "name": "My New Pack",
  "cat": ["funny"],
  "regions": ["PK"]
}

# 3. Bump "v" in _master.json

# 4. Push to main
git add . && git commit -m "add my-new-pack" && git push
```

CI automatically: validates в†’ generates region JSONs в†’ deploys to GitHub Pages.

### 2. Add a new category

```json
// In _master.json в†’ categories[]
{ "id": "memes", "name": "Memes" }

// Then add "memes" to cat[] of relevant packs
// Bump "v" в†’ push
```

### 3. Add a new country/region

```json
// In _master.json в†’ regions[]
"MA"

// Add "MA" to regions[] of packs for Morocco
// Bump "v" в†’ push в†’ CI creates regions/MA.json
```

### 4. Make a pack global

```json
"regions": ["*"]
// Shows in every region, every zone, _default
```

## Image Rules

| Rule | Spec |
|---|---|
| Tray icon | `tray_icon.webp`, 96Г—96px recommended |
| Sticker naming | `1.webp`, `2.webp`, `3.webp` ... sequential, no gaps |
| Max per pack | 30 stickers |
| Format | WebP with transparency |
| Size | 512Г—512px recommended, under 100KB each |

## API Endpoints (for Android app)

Base URL: `https://hafizg.github.io/ai_sticker_maker`

| Endpoint | Description |
|---|---|
| `/index.json` | App bootstrap (categories, regions, zones, version) |
| `/regions/PK.json` | Packs for Pakistan |
| `/regions/_arab.json` | Packs for Arab zone fallback |
| `/regions/_default.json` | Global fallback |
| `/packs/{id}/tray_icon.webp` | Pack tray icon |
| `/packs/{id}/{n}.webp` | Sticker image (n = 1..count) |

### Fallback Chain (app-side)

```
User country в†’ PK в†’ fetch regions/PK.json  вњ…
User country в†’ LB в†’ not in regions в†’ check zones в†’ _arab в†’ fetch regions/_arab.json  вњ…
User country в†’ JP в†’ not in regions, not in zones в†’ fetch regions/_default.json  вњ…
```

## _master.json Reference

```json
{
  "v": 1,                          // в†ђ bump on every change
  "categories": [                   // в†ђ tabs shown in app
    { "id": "funny", "name": "Funny" }
  ],
  "regions": ["PK", "IN", "SA"],   // в†ђ countries with dedicated files
  "zones": [                        // в†ђ fallback groups
    { "id": "_arab", "countries": ["LB", "SY"] }
  ],
  "packs": [                        // в†ђ sticker packs
    {
      "id": "pack-folder-name",     // в†ђ must match folder in packs/
      "name": "Display Name",       // в†ђ shown in app
      "cat": ["funny", "religious"],// в†ђ 1+ category IDs
      "regions": ["PK", "*"]        // в†ђ country codes, zone IDs, or "*"
    }
  ]
}
```

### regions[] values

| Value | Meaning |
|---|---|
| `"PK"` | Pakistan only |
| `"_arab"` | Arab zone (all countries in that zone) |
| `"*"` | Every region + every zone + `_default` |
| `"_default"` | Only in `_default.json` |
